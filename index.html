The best way to manage Ruby applications

Getting Started

Getting started with bundler is easy 
$ gem install bundler
Specify your dependencies in a Gemfile in your project's root 
source :rubygems
gem "nokogiri"
Install all of the required gems from your specified sources 
$ bundle install
$ git add Gemfile
Inside your app, load up the bundled environment 
require "rubygems"
require "bundler"
Bundler.setup

require "nokogiri"
# any other requires you might need
When you're ready to share your application or deploy it, lock down all the versions of the gems you're using 
$ bundle lock
$ git add Gemfile.lock

Checking Out an Application With a Gemfile

Install the required gems 
$ bundle install

Digging Further

Store all of the required gems in your application. All future installs will get gems from this cache, bypassing rubygems.org 
$ bundle package
Put dependencies in a group, so they can be ignored at install time or required together in your application Learn more: Groups 
group :development do
  gem "wirble"
end
Use a gem that is stored in git and has a .gemspec at its root. Gemspecs with bins or C extensions are supported Learn more: Git 
gem "nokogiri", :git =>
  "git://github.com/tenderlove/nokogiri.git"
Use a gem that you are actively developing on your file system 
gem "nokogiri", :path => "~/Code/nokogiri"
Install gems to a different location (the default is ~/.bundle) 
$ bundle install vendor --disable-shared-gems
